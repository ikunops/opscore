# OpsCore 变更概览 — 防火墙并入网络 + 资源模块增强

> 运行地址: http://localhost:8080  (二进制已重建并重启)

## 1. 防火墙并入「防火墙和网络」模块
- 侧栏原「网络」改名为 **防火墙和网络**;原独立「防火墙」侧栏项删除。
- 进入模块后顶部是两个同级 tab:**网络 / 防火墙**。
  - 点击 **网络**: 回到原来的双卡界面(网络接口 + 监听端口并排显示)。
  - 点击 **防火墙**: 渲染原来完整的防火墙界面(含状态、启动/停止/重启、端口开关、IP 黑白名单、现有规则、审计链、高危确认弹窗)。
- 后端 `/api/core/firewall*` 接口不变。

## 2. 系统资源模块修复与增强(保留原有内容)
- **修 CPU 仪表盘 `V%` bug**:gauge `detail.formatter` 已修正, 现在使用两位小数百分比。
- **实时趋势保留原状**: 默认显示原来的 **CPU% + 网络上下行** 组合图, 与之前一致。
- **新增指标切换**: 在「综合」基础上新增 CPU / 内存 / Swap / 网络 四个选项, 不会替换原有综合图。
- **新增时间窗**: 5 分钟 / 15 分钟 / 1 小时切换, 默认 5 分钟, 作用于实时趋势图。
- **Swap 迷你趋势**: 系统负载卡片里的 Swap 区新增一段实时 sparkline。

## 3. 构建 / 重启
- `cd web && npm run build` → `go build -o opscore.exe .`。
- 杀旧进程后重启, `:8080` 验证通过(manifest 正确、前端 200)。
